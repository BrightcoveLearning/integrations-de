    Einbetten von Brightcove-Spielern in ein CMS

    Dieses Thema enthält Anleitungen zum Bereitstellen eines Dialogfelds für CMS-Benutzer, mit dem sie einen Brightcove Player in eine Seite einbetten können.

    Einführung

    Wenn Ihre CMS-Benutzer einen Video- oder Playlist-Player in eine Seite einbetten, möchten Sie ihnen bei der Generierung des Brightcove-Player-Einbettungscodes so viel Flexibilität wie möglich bieten, damit sie das Erscheinungsbild des Brightcove Players auf der Webseite steuern können, aber nicht erwarten, dass sie HTML- oder JavaScript-Experten sind.

    In diesem Thema finden Sie Richtlinien, wie das geht. Die Zielgruppe für dieses Dokument besteht hauptsächlich aus Brightcove-Partnern, ist jedoch für jeden nützlich, der Video Cloud oder den eigenständigen Brightcove Player mit Ihrem CMS integriert.

    Spieler bettet Dialogverhalten ein

    Es sollte einen Dialog geben, der es dem Benutzer ermöglicht, die Formatierung des Players auf der Webseite zu steuern. Dieser Dialog sollte es dem Benutzer ermöglichen, die unten aufgeführten Felder festzulegen. Es wird davon ausgegangen, dass der Benutzer bereits ein Brightcove-Konto und ein Video oder eine Playlist ausgewählt hat, bevor dieser Dialog angezeigt wird. Andernfalls sollte die Auswahl des Brightcove-Kontos und des Videos oder der Wiedergabeliste zuerst erfolgen, wie in Anhang A unten beschrieben.

    1. Spieler: Der Benutzer sollte in der Lage sein, einen Spieler aus den verfügbaren Brightcove Spielern auszuwählen. Wenn sie einen Videoplayer einbetten, sollten nur Spieler ohne Wiedergabelisten zur Auswahl stehen. Wenn sie einen Playlist-Spieler einbetten, sollten nur Playlist-fähige Spieler zur Auswahl stehen.
    2. Autoplay: Der Benutzer sollte Autoplay aktivieren/deaktivieren können. Die Standardeinstellung ist deaktiviert.
    3. Stummschalten: Der Benutzer sollte in der Lage sein, Stummschaltung zu aktivieren/deaktivieren. Die Standardeinstellung ist deaktiviert.
    4. Typ einbetten: Der Benutzer sollte in der Lage sein, zwischen iFrame- und Javascript-Einbettungscode zu wählen.
      1. Für einen Videoplayer sollte der Standardwert Javascript sein.
      2. Für einen Playlist-Player sollte der Standardwert iFrame sein.
    5. Größe: Der Benutzer sollte in der Lage sein, zwischen Responsive[1-1] oder Feste Größe[1-2]zu wählen.
      1. Für einen Videoplayer sollte der Standardwert Responsive sein.
      2. Für einen Playlist-Spieler:
        1. Bei iFrame sollte der Standardwert Responsive sein.
        2. Bei Javascript sollte der Standardwert Fix sein.
    6. Seitenverhältnis: Der Benutzer sollte in der Lage sein, das Seitenverhältnis[1-1]auszuwählen.
      1. Folgende Optionen stehen zur Auswahl:
        • 16:9
        • 4:3
        • Custom
      2. Die Standardeinstellung sollte sein 16:9.
    7. Width, Height:DerBenutzer sollte in der Lage sein, Breite und Höhe [1-1] einzugeben.
      1. Wenn das Seitenverhältnis 16:9 oder 4:3 beträgt, sollte das Feld Höhe ausgegraut und automatisch basierend auf Breite und Seitenverhältnisberechnet werden.
      2. Die Standardwerte sollten 640 für Breite und 360 Höhe sein, wenn das Seitenverhältnis Benutzerdefiniert ist.

    Hinweise

    • [ 1-1] Wenn das CMS einen nativen Container zum Einbetten von Code bereitstellt, mit dem der Benutzer Responsive vs Fest und Width/Height angeben kann, ist es möglicherweise sinnvoller, den Container für die Dimensionierung zu verwenden und den Brightcove-Einbettungscode immer reaktionsfähig zu machen.
    • [ 1-2] Der unten enthaltene Responsive-Einbettungscode verwendet die Breite als maximale Breite.

    Einbetten von Parametern und Code

    In den folgenden Abschnitten wird beschrieben, wie der Einbettungscode des Brightcove-Spielers basierend auf der Benutzerauswahl generiert wird. Die %XYZ% Werte werden wie unten beschrieben in den Einbettungscode ersetzt.

    Gängige Parameter

    • %AUTOPLAY% = Autoplay wenn Autoplay aktiviert, sonst ist es eine null Zeichenfolge
    • %MUTED% = stummgeschaltet wenn Muted aktiviert, sonst ist es eine null Zeichenfolge
    • %ACCOUNTID% = Benutzer hat die Brightcove-Konto-ID ausgewählt
    • %VIDEOID% = Vom Benutzer ausgewählte Video-ID
    • %PLAYLISTID% = Vom Benutzer ausgewählte Wiedergabelisten-ID
    • %PLAYERID% = Vom Benutzer ausgewählte Videoplayer-ID oder Playlist-Spieler-ID
    • %CMS% = Name des CMS
    • %CMSVERSION% = Version von CMS
    • %CONNECTORVERSION% = Version des Steckers

    Beispiel Implementierung

    Anhang A

    Wenn vor dem Öffnen des Player-Einbettungsdialogs kein Video oder eine Wiedergabeliste ausgewählt wird, sollte das Video oder die Wiedergabeliste (und ggf. das Brightcove-Konto) im Dialogfeld ausgewählt werden. Der Dialog sollte dem Benutzer die unten aufgeführten Felder zeigen:

    1. Brightcove Konto: Der Benutzer sollte in der Lage sein, ein Brightcove-Konto zu wählen, wenn es noch nicht ausgewählt ist.
    2. Suche filtern: Der Benutzer sollte in der Lage sein, eine Suchzeichenfolge einzugeben, um die Liste der angezeigten Videos oder Wiedergabelisten zu filtern. Wenn Sie die Brightcove-Such-API verwenden, sollte die Suchzeichenfolge URI-codiert sein (siehe Beispiel unten).
    3. Mappe: Der Benutzer sollte in der Lage sein, einen Ordnernamen aus dem Brightcove-Konto auszuwählen, um die Liste der angezeigten Videos zu filtern. Die Brightcove API bietet keinen integrierten Filter für Ordner, daher muss der Client-Code alle Videos abrufen und dann nach Ordner filtern. (Gilt nicht für Playlists)
    4. Begrenzung: Der Benutzer sollte in der Lage sein, die Anzahl der zurückgegebenen Videos zu begrenzen, hauptsächlich um die Leistung der Suche zu verbessern. Bei Verwendung der Brightcove-Suche beträgt das maximale Limit, das angegeben werden kann, 100. Um mehr als 100 Videos zurückzugeben, muss ein Paging-Mechanismus implementiert werden. Wenn der Benutzer einen Ordner zum Filtern ausgewählt hat, muss der Kunde außerdem alle Videos im Konto mithilfe des Paging-Mechanismus anfordern und diese Liste dann lokal nach Ordner filtern und die durch das Benutzerlimit angegebene Anzahl von Videos zurückgeben. (Gilt nicht für Playlists)
    5. Sortiert nach: Der Benutzer sollte in der Lage sein, ein Sortierfeld auszuwählen.
      1. Bei Videos sollte der Benutzer in der Lage sein, den Videonamen, das Aktualisierungsdatum, das Erstellungsdatum, das Startdatum und die Gesamtwiedergabe auszuwählen. Der Standardwert sollte das Aktualisierungsdatum sein.
      2. Bei Wiedergabelisten wählt der Benutzer entweder den Namen und das Änderungsdatum. Der Standardwert sollte das Änderungsdatum sein.
    6. Sortierreihenfolge: Der Benutzer sollte in der Lage sein, eine aufsteigende oder absteigende Sortierreihenfolge auszuwählen. Der Standardwert sollte absteigend sein.

    Der Dialog sollte folgendes Verhalten haben:

    1. Zeigen Sie eine Liste von Videos oder Wiedergabelisten basierend auf der obigen Benutzerauswahl an. Erlauben Sie dem Benutzer, ein Video oder eine Wiedergabeliste auszuwählen.
    2. Wenn Sie eine Liste von Videos anzeigen:
      1. Es sollten nur aktive Videos aufgeführt werden.
      2. Das Miniaturbild, der Videoname und die ID sollten angezeigt werden.
      3. Es kann nur ein Video ausgewählt werden.
    3. Wenn Sie eine Liste von Wiedergabelisten anzeigen:
      1. Wenn es sich bei der Wiedergabeliste um eine manuelle Wiedergabeliste handelt, zeigen Sie den Namen der Wiedergabeliste, die ID und die Anzahl der Videos in der Wiedergabeliste an.
      2. Wenn es sich bei der Wiedergabeliste um eine intelligente Wiedergabeliste handelt, zeigen Sie den Namen der Wiedergabeliste, die ID und - anstelle der Anzahl der Videos an.
      3. Es kann nur eine Playlist ausgewählt werden.
    4. Zeigen Sie einen anklickbaren Link an, der den Video- oder Playlist-Player in einem neuen Browser-Tab öffnet.

    Beispiele für URI-Codierung

    1. Zeichenfolge suchen: great+blue

      String in CMS API Call sollte sein: great%2Bblue

      Videos zurückgegeben: alles was die Wörter enthält: great UND blue[2-1]

    2. Zeichenfolge suchen: great blue

      String in CMS API Call sollte sein: great%20blue

      Die zurückgegebenen Videos enthalten alle die Wörter: great ODER blue[2-1]

    3. Zeichenfolge suchen: "great blue"

      String in CMS API Call sollte sein: %22great%20blue%22

      Die zurückgegebenen Videos enthalten nur den vollständigen Satz: great blue[2-1]

    Hinweise

    • [ 2-1] Stemming gilt in allen Fällen. Suchen wie die oben gezeigten würden auch mit den Great Blues betitelten Videos übereinstimmen Great Blue Heron.
