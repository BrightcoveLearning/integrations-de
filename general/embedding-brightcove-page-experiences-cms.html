Einführung

Zusätzlich zu den grundlegenden Einzelvideoplayern und Wiedergabelisten-Playern bietet Brightcove einen Multi-Video-Player-Typ namens In-Page Experience. Sie möchten Benutzern das einfache Einbetten einer Experience ermöglichen und so viel Flexibilität wie möglich bei der Generierung des Brightcove Experience-Einbettungscodes bieten, damit sie das Erscheinungsbild steuern können, ohne HTML- oder JavaScript-Experten zu sein.

Beachten Sie, dass In-Page-Erlebnisse Teil der Galerie sind, die in der Video Marketing Suite und Enterprise Video Suite und kann auch als Add-On-Paket für Video Cloud erworben werden. Wenden Sie sich an Ihren Account Manager, um weitere Informationen zu erhalten.

In-Page-Auswahldialog für die Erfahrung

Es sollte ein Dialogfeld vorhanden sein, in dem der Benutzer eine In-Page-Erfahrung auswählen kann. In diesem Dialogfeld sollte der Benutzer die folgenden Felder festlegen können:

  1. Brightcove-Konto: Der Benutzer sollte in der Lage sein, ein Brightcove-Konto auszuwählen, falls noch keines ausgewählt ist.
  2. Suchfilter: Der Benutzer sollte in der Lage sein, eine Suchzeichenfolge einzugeben, um die Liste der angezeigten Erfahrungen zu filtern. Wenn Sie die Brightcove-Such-API verwenden, sollte die Suchzeichenfolge URI-codiert sein.
  3. Auftrag: Die Liste der angezeigten Erfahrungen sollte nach Namen sortiert sein. Der Benutzer sollte in der Lage sein, eine aufsteigende oder absteigende Sortierreihenfolge auszuwählen. Die Standardeinstellung sollte aufsteigend sein.

Der Dialog sollte das folgende Verhalten haben:

  1. Zeigen Sie eine Liste der Erfahrungen basierend auf der Benutzerauswahl oben an. Ermöglichen Sie dem Benutzer, eine Erfahrung auszuwählen.
  2. Für die angezeigte Liste der Erfahrungen:
    1. Es sollten nur veröffentlichte Erfahrungen angezeigt werden
    2. Der Name, die Vorlage und die ID der Erfahrung sollten angezeigt werden.
  3. Zeigen Sie einen anklickbaren Link an, der den Experience Player in einem neuen Browser-Tab öffnet.

Verwandte Brightcove-Dokumentation

  • Integration Ihres CMS in die Video Cloud
  • Liste der Erfahrungen abrufen
    • Nach Namen in aufsteigender Reihenfolge sortieren: ?sort=name
    • Nach Namen in absteigender Reihenfolge sortieren: ?sort=-name
    • Suche: ?q={searchstring}
  • Holen Sie sich Details für eine In-Page-Erfahrung

Beispiel für eine Dialogimplementierung

In-Page-Dialogfeld zum Einbetten von Erfahrungen

Es sollte ein Dialogfeld vorhanden sein, in dem der Benutzer die Formatierung der In-Page-Erfahrung auf der Webseite steuern kann. In diesem Dialogfeld sollte der Benutzer die folgenden Felder festlegen können:

  1. Einbettungstyp: Der Benutzer sollte in der Lage sein, zwischen iFrame- und Javascript-Einbettungscode zu wählen. Der Standardwert sollte Javascript sein.
  2. Größe: Der Benutzer sollte zwischen Responsive oder Fixed Sizing wählen können.[1-1]
    1. Bei iFrame sollte der Standardwert "Fest" und "Reaktionsschnell" deaktiviert sein.
    2. Bei Javascript sollte der Standardwert Responsive sein.
  3. Breite Höhe: Der Benutzer sollte in der Lage sein, Breite und Höhe einzugeben.[1-1]
    1. Bei iFrame sollte der Standardwert "Fest" und "Reaktionsschnell" deaktiviert sein.
    2. Wenn Sie reagieren, sollten Breite und Höhe deaktiviert sein.
  4. Erfahrungsvideos überschreiben: Der Benutzer sollte in der Lage sein, die Standardvideos in einer Erfahrung mit einer Liste von Videos oder mit einer Wiedergabeliste zu überschreiben. Wenn der Benutzer die Überschreibung der Erfahrungsvideos auswählt, können Sie dem Benutzer erlauben, Videos oder eine Wiedergabeliste im Abschnitt "Überschreiben von Erfahrungsvideos" unten auszuwählen.

Der Dialog sollte das folgende zusätzliche Verhalten haben:

  1. Der generierte Brightcove-Einbettungscode sollte dem Benutzer im Dialogfeld angezeigt werden.
  2. Der Benutzer sollte in der Lage sein, Änderungen am Einbettungscode vorzunehmen. Obwohl wir versuchen, dem Benutzer so viel Flexibilität wie möglich bei der Konfiguration des Einbettungscodes zu geben, kann es Fälle geben, in denen er überschreiben muss, was automatisch generiert wird.
  3. Wenn der Benutzer eine der vorherigen Auswahlen im Dialogfeld ändert, werden die Änderungen des Benutzers durch neu generierten Code überschrieben.

Anmerkungen

  • [1-1] Wenn der CMS einen nativen Container zum Einbetten von Code bereitstellt, der es dem Benutzer ermöglicht, Responsive vs Fixed und Width/Height anzugeben, ist es möglicherweise sinnvoller, den Container für die Größenanpassung zu verwenden und den Brightcove-Einbettungscode immer responsive zu machen.

Überschreiben von Erfahrungsvideos

Wenn der Benutzer die Experience-Videos überschreiben möchte, sollte der Dialog es dem Benutzer ermöglichen, die unten aufgeführten Felder festzulegen:

  1. Suchfilter: Der Benutzer sollte in der Lage sein, eine Suchzeichenfolge einzugeben, um die Liste der angezeigten Videos oder Wiedergabelisten zu filtern. Wenn Sie die Brightcove-Such-API verwenden, sollte die Suchzeichenfolge URI-codiert sein.
  2. Mappe: Der Benutzer sollte in der Lage sein, einen Ordnernamen aus dem Brightcove-Konto auszuwählen, um die Liste der angezeigten Videos zu filtern. Die Brightcove-API bietet keinen integrierten Filter für Ordner, sodass der Clientcode alle Videos abrufen und dann nach Ordner filtern muss. (Gilt nicht für Wiedergabelisten)
  3. Grenze: Der Benutzer sollte in der Lage sein, die Anzahl der zurückgegebenen Videos zu begrenzen, um vor allem die Leistung der Suche zu verbessern. Bei Verwendung der Brightcove-Suche kann maximal 100 angegeben werden. Um mehr als 100 Videos zurückzugeben, muss ein Paging-Mechanismus implementiert werden. Wenn der Benutzer einen zu filternden Ordner ausgewählt hat, muss der Client alle Videos im Konto mithilfe des Paging-Mechanismus anfordern und diese Liste dann lokal nach Ordner filtern und die vom Benutzerlimit angegebene Anzahl von Videos zurückgeben. (Gilt nicht für Wiedergabelisten)
  4. Sortieren nach: Der Benutzer sollte in der Lage sein, ein Sortierfeld auszuwählen.
    1. Bei Videos sollte der Benutzer in der Lage sein, den Videonamen, das Aktualisierungsdatum, das Erstellungsdatum, das Startdatum und die Gesamtwiedergabe auszuwählen. Der Standardwert sollte das Aktualisierungsdatum sein.
    2. Für Wiedergabelisten wählt der Benutzer entweder den Namen und das Änderungsdatum. Das Standarddatum sollte geändert werden.
  5. Sortierung: Der Benutzer sollte in der Lage sein, eine aufsteigende oder absteigende Sortierreihenfolge auszuwählen. Die Standardeinstellung sollte absteigend sein.

Der Dialog sollte das folgende Verhalten haben:

  1. Zeigen Sie eine Liste von Videos oder Wiedergabelisten basierend auf der Benutzerauswahl oben an. Ermöglichen Sie dem Benutzer, mehrere Videos oder eine Wiedergabeliste auszuwählen.
  2. Beim Anzeigen einer Liste von Videos:
    1. Es sollten nur aktive Videos aufgelistet werden.
    2. Das Miniaturbild, der Videoname und die ID sollten angezeigt werden.
    3. Ein oder mehrere Videos können ausgewählt werden.
  3. Beim Anzeigen einer Liste von Wiedergabelisten:
    1. Wenn es sich bei der Wiedergabeliste um eine manuelle Wiedergabeliste handelt, zeigen Sie den Namen der Wiedergabeliste, die ID und die Anzahl der Videos in der Wiedergabeliste an.
    2. Wenn es sich bei der Wiedergabeliste um eine intelligente Wiedergabeliste handelt, zeigen Sie den Namen der Wiedergabeliste, die ID und --- anstelle der Anzahl der Videos an.
    3. Es kann nur eine Wiedergabeliste ausgewählt werden.
  4. Zeigen Sie einen anklickbaren Link an, der den Video- oder Wiedergabelisten-Player in einem neuen Browser-Tab öffnet.

Parameter und Code einbetten

In diesem Abschnitt wird beschrieben, wie Sie den Experience-Einbettungscode basierend auf der Benutzerauswahl generieren. Das %XYZ% Werte werden wie unten beschrieben in den Einbettungscode eingesetzt.

Gemeinsame Parameter

  • %ACCOUNTID% = Vom Benutzer ausgewählte Brightcove-Konto-ID
  • %VIDEOIDLIST% = Vom Benutzer ausgewählte Video-IDs, durch Kommas getrennt ohne Leerzeichen
  • %PLAYLISTID% = Vom Benutzer ausgewählte Wiedergabelisten-ID
  • %EXPERIENCEID% = Vom Benutzer ausgewählte Video-Player-ID oder Playlist-Player-ID
  • %CMS% = Name des CMS
  • %CMSVERSION% = Version von CMS
  • %CONNECTORVERSION% = Version des Steckers
