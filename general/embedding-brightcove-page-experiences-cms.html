    Einbetten von In-Page-Erlebnissen in ein CMS

    Dieses Thema enthält Richtlinien für die Integration von Brightcove In-Page Experiences in Ihr CMS.

    Einführung

    Zusätzlich zu den grundlegenden Einzelvideoplayern und Playlist-Playern bietet Brightcove einen Multi-Video-Player-Typ namens In-Page Experience. Sie möchten es Benutzern ermöglichen, ein Experience einfach einzubetten und so viel Flexibilität wie möglich bei der Generierung des Brightcove Experience-Einbettungscodes zu bieten, damit sie das Erscheinungsbild steuern können, ohne HTML- oder JavaScript-Experten zu sein.

    Beachten Sie, dass In-Page Experiences Teil der Gallery sind, die in der Video Marketing Suite und Enterprise Video Suite enthalten ist und auch als Add-On-Paket für Video Cloud erworben werden kann. Ihr Kontomanager teilt Ihnen gerne weitere Details hierzu mit.

    Dialog zur In-Page-Erfahrungsauswahl

    Es sollte einen Dialog geben, der es dem Benutzer ermöglicht, ein In-Page Experience zu wählen. Dieser Dialog sollte es dem Benutzer ermöglichen, die unten aufgeführten Felder festzulegen:

    1. Brightcove Konto: Der Benutzer sollte in der Lage sein, ein Brightcove-Konto zu wählen, wenn es noch nicht ausgewählt ist.
    2. Suche filtern: Der Benutzer sollte in der Lage sein, eine Suchzeichenfolge einzugeben, um die Liste der angezeigten Erlebnisse zu filtern. Wenn Sie die Brightcove-Such-API verwenden, sollte die Suchzeichenfolge URI-codiert sein.
    3. BESTELLEN: Die Liste der angezeigten Experiences sollte nach Namen sortiert sein. Der Benutzer sollte in der Lage sein, eine aufsteigende oder absteigende Sortierreihenfolge auszuwählen. Der Standardwert sollte aufsteigend sein.

    Der Dialog sollte folgendes Verhalten haben:

    1. Zeigen Sie eine Liste von Erlebnissen basierend auf der obigen Benutzerauswahl an. Erlauben Sie dem Benutzer, eine Erfahrung auszuwählen.
    2. Für die angezeigte Liste von Erlebnissen:
      1. Es sollten nur veröffentlichte Experiences angezeigt werden
      2. Der Name, die Vorlage und die ID der Erfahrung sollten angezeigt werden.
    3. Zeigen Sie einen anklickbaren Link an, der den Experience Player in einem neuen Browser-Tab öffnet.

    Verwandte Brightcove Dokumentation

    • Integrieren Sie Ihr CMS mit Video Cloud
    • Liste der Erlebnisse abrufen
      • Sortieren nach Namen in aufsteigender Reihenfolge: ?sort=name
      • Sortieren nach Namen in absteigender Reihenfolge: ?sort=-name
      • Suche: ?q={searchstring}
    • Details für ein In-Page-Erlebnis abrufen

    Beispiel für Dialog-Implementierung

    Dialog zum Einbetten von In-Page Ex

    Es sollte einen Dialog geben, der es dem Benutzer ermöglicht, die Formatierung der In-Page Experience auf der Webseite zu steuern. Dieser Dialog sollte es dem Benutzer ermöglichen, die unten aufgeführten Felder festzulegen:

    1. Typ einbetten: Der Benutzer sollte in der Lage sein, zwischen iFrame- und Javascript-Einbettungscode zu wählen. Der Standardwert sollte Javascript sein.
    2. Größe: Der Benutzer sollte in der Lage sein, zwischen Responsive oder Fixed Sizing zu wählen.[ 1-1]
      1. Bei iFrame sollte der Standardwert Fix und Responsive deaktiviert sein.
      2. Bei Javascript sollte der Standardwert Responsive sein.
    3. Breite, Höhe: Der Benutzer sollte in der Lage sein, Breite und Höhe einzugeben.[ 1-1]
      1. Bei iFrame sollte der Standardwert Fix und Responsive deaktiviert sein.
      2. Wenn Responsive, sollten die Breite und Höhe deaktiviert werden.
    4. Erfahrungsvideos überschreiben: Der Benutzer sollte in der Lage sein, die Standardvideos in einem Erlebnis mit einer Liste von Videos oder mit einer Playlist zu überschreiben. Wenn der Benutzer die Überschreibungen der Erlebnisvideos auswählt, gestatten Sie dem Benutzer, Videos oder eine Wiedergabeliste gemäß dem Abschnitt „Übergeordnete Erlebnisvideos“ unten auszuwählen.

    Der Dialog sollte das folgende zusätzliche Verhalten aufweisen:

    1. Der generierte Brightcove Einbettungscode sollte dem Benutzer im Dialog angezeigt werden.
    2. Der Benutzer sollte in der Lage sein, Änderungen am Einbettungscode vorzunehmen. Obwohl wir versuchen, dem Benutzer bei der Konfiguration des Einbettungscodes so flexibel wie möglich zu geben, kann es Fälle geben, in denen er überschreiben muss, was automatisch generiert wird.
    3. Wenn der Benutzer eine der vorherigen Auswahlen im Dialog ändert, werden die Änderungen des Benutzers durch neu generierten Code überschrieben.

    Hinweise

    • [ 1-1] Wenn das CMS einen nativen Container zum Einbetten von Code bereitstellt, mit dem der Benutzer Responsive vs Fest und Width/Height angeben kann, ist es möglicherweise sinnvoller, den Container für die Dimensionierung zu verwenden und den Brightcove-Einbettungscode immer reaktionsfähig zu machen.

    Überschreiben von Erlebnisvideos

    Wenn der Benutzer die Experience-Videos überschreiben möchte, sollte der Dialog es dem Benutzer ermöglichen, die unten aufgeführten Felder festzulegen:

    1. Suche filtern: Der Benutzer sollte in der Lage sein, eine Suchzeichenfolge einzugeben, um die Liste der angezeigten Videos oder Wiedergabelisten zu filtern. Wenn Sie die Brightcove-Such-API verwenden, sollte die Suchzeichenfolge URI-codiert sein.
    2. Mappe: Der Benutzer sollte in der Lage sein, einen Ordnernamen aus dem Brightcove-Konto auszuwählen, um die Liste der angezeigten Videos zu filtern. Die Brightcove API bietet keinen integrierten Filter für Ordner, daher muss der Client-Code alle Videos abrufen und dann nach Ordner filtern. (Gilt nicht für Playlists)
    3. Begrenzung: Der Benutzer sollte in der Lage sein, die Anzahl der zurückgegebenen Videos zu begrenzen, hauptsächlich um die Leistung der Suche zu verbessern. Bei Verwendung der Brightcove-Suche beträgt das maximale Limit, das angegeben werden kann, 100. Um mehr als 100 Videos zurückzugeben, muss ein Paging-Mechanismus implementiert werden. Wenn der Benutzer einen Ordner zum Filtern ausgewählt hat, muss der Kunde außerdem alle Videos im Konto mithilfe des Paging-Mechanismus anfordern und diese Liste dann lokal nach Ordner filtern und die durch das Benutzerlimit angegebene Anzahl von Videos zurückgeben. (Gilt nicht für Playlists)
    4. Sortiert nach: Der Benutzer sollte in der Lage sein, ein Sortierfeld auszuwählen.
      1. Bei Videos sollte der Benutzer in der Lage sein, den Videonamen, das Aktualisierungsdatum, das Erstellungsdatum, das Startdatum und die Gesamtwiedergabe auszuwählen. Der Standardwert sollte das Aktualisierungsdatum sein.
      2. Bei Wiedergabelisten wählt der Benutzer entweder den Namen und das Änderungsdatum. Der Standardwert sollte das Änderungsdatum sein.
    5. Sortierreihenfolge: Der Benutzer sollte in der Lage sein, eine aufsteigende oder absteigende Sortierreihenfolge auszuwählen. Der Standardwert sollte absteigend sein.

    Der Dialog sollte folgendes Verhalten haben:

    1. Zeigen Sie eine Liste von Videos oder Wiedergabelisten basierend auf der obigen Benutzerauswahl an. Erlauben Sie dem Benutzer, mehrere Videos oder eine Wiedergabeliste auszuwählen.
    2. Wenn Sie eine Liste von Videos anzeigen:
      1. Es sollten nur aktive Videos aufgeführt werden.
      2. Das Miniaturbild, der Videoname und die ID sollten angezeigt werden.
      3. Es können ein oder mehrere Videos ausgewählt werden.
    3. Wenn Sie eine Liste von Wiedergabelisten anzeigen:
      1. Wenn es sich bei der Wiedergabeliste um eine manuelle Wiedergabeliste handelt, zeigen Sie den Namen der Wiedergabeliste, die ID und die Anzahl der Videos in der Wiedergabeliste an.
      2. Wenn es sich bei der Wiedergabeliste um eine intelligente Wiedergabeliste handelt, zeigen Sie den Namen der Wiedergabeliste, die ID und — anstelle der Anzahl der Videos an.
      3. Es kann nur eine Playlist ausgewählt werden.
    4. Zeigen Sie einen anklickbaren Link an, der den Video- oder Playlist-Player in einem neuen Browser-Tab öffnet.

    Einbetten von Parametern und Code

    Der Abschnitt beschreibt, wie der Experience-Einbettungscode basierend auf der Benutzerauswahl generiert wird. Die %XYZ% Werte werden wie unten beschrieben in den Einbettungscode ersetzt.

    Gängige Parameter

    • %ACCOUNTID% = Benutzer hat die Brightcove-Konto-ID ausgewählt
    • %VIDEOIDLIST% = Vom Benutzer ausgewählte Video-IDs, kommageteitig ohne Leerzeichen
    • %PLAYLISTID% = Vom Benutzer ausgewählte Wiedergabelisten-ID
    • %EXPERIENCEID% = Vom Benutzer ausgewählte Videoplayer-ID oder Playlist-Spieler-ID
    • %CMS% = Name des CMS
    • %CMSVERSION% = Version von CMS
    • %CONNECTORVERSION% = Version des Steckers
