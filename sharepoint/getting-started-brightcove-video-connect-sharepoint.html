Installieren von Brightcove Video Connect für SharePoint

Lesen Sie vor Beginn der Installation die folgenden Informationen zu den Voraussetzungen und Berechtigungen:

  • Bevor ein Benutzer eine App für SharePoint hinzufügen kann, muss ein Mitglied der Gruppe Farmadministratoren die Umgebung so konfigurieren, dass Apps für SharePoint unterstützt werden. Weitere Informationen finden Sie unter Konfigurieren Sie eine Umgebung für Apps für SharePoint 2013 oder SharePoint 2016 oder SharePoint 2019.
  • Ein Benutzer muss die haben Website verwalten und Erstellen Sie Unterseiten Berechtigungen zum Hinzufügen einer App für SharePoint. Standardmäßig sind diese Berechtigungen nur für Benutzer verfügbar, die über die Berechtigungsstufe "Vollzugriff" verfügen oder zur Gruppe der Websitebesitzer gehören.
  • Wenn ein Benutzer eine App für SharePoint hinzufügt, fordert die App Berechtigungen an, die sie zum Funktionieren benötigt (z. B. Zugriff auf die Suche oder Erstellen einer Liste). Benutzer, die nicht über diese Berechtigungen verfügen, werden darüber informiert, dass sie nicht über ausreichende Berechtigungen verfügen und die App nicht hinzugefügt werden kann. Der Benutzer kann sich an einen Site- oder Farmadministrator wenden, um zu prüfen, ob der Administrator die App hinzufügen kann.
  • Ein Benutzer, der als Systemkonto an einer Site angemeldet ist, kann keine App installieren. Das Systemkonto kann keine App-Lizenzen importieren, da dies zu Leistungsproblemen führen kann.

Installieren der Brightcove Video Connect-App auf einer Site

Führen Sie die folgenden Schritte aus, um den Brightcove-Connector auf einer Site zu installieren:

  1. Stellen Sie sicher, dass das Benutzerkonto, das dieses Verfahren ausführt, Mitglied der Gruppe Site Owners ist.
  2. Die App kann auf zwei Arten zu einer Site hinzugefügt werden:
    1. Auf jeder Seite, von der Einstellungen Menüauswahl Eine App hinzufügen Möglichkeit.
    2. Von Inhalt der Website Verwendung der Eine App hinzufügen Möglichkeit.
  3. Klicken Sie anschließend auf "Von Ihrer Organisation" und dann auf das BrightcoveConnector App
  4. Klicken Sie im Dialogfeld "Berechtigung erteilen" auf "Vertrauen".
  5. Zu diesem Zeitpunkt ist die BrightcoveConnector App für SharePoint wurde hinzugefügt und wird im Abschnitt "Apps" Ihrer Website-Inhaltsliste angezeigt. Es ist jetzt zur Verwendung auf der Site verfügbar.

Installieren des Player-Lösungspakets für SharePoint Online

Mit dem im Brightcove Video Connect-Paket enthaltenen Brightcove Player-Lösungspaket kann der Benutzer Brightcove-Videoplayer in SharePoint Site-Seiten einbetten. Das Brightcove Player-Lösungspaket verwendet sofort einsatzbereite SharePoint-Steuerelemente, die sowohl mit SharePoint Online als auch mit SharePoint Server kompatibel sind und daher mit beiden verwendet werden können. Eine Player-Webpart-Sandbox-Lösung zum Einbetten von Videos ist auch im Brightcove Video Connect-Paket verfügbar, kann jedoch nur mit SharePoint Server vor Ort verwendet werden, da SharePoint Online und SharePoint Server, die von Microsoft gehostet werden, Sicherheitsbeschränkungen aufweisen, die Sandbox-Lösungen verbieten. Diese Lösung ist im nächsten Abschnitt beschrieben.

Um einen Brightcove-Player in SharePoint Online einzubetten, kann der Benutzer mit dem hier beschriebenen Lösungspaket einen Video- oder Wiedergabelisten-Player auswählen und dann Der Player-Einbettungscode wird generiert, damit der Benutzer ihn in das Dialogfeld SharePoint-Einbettungscode kopieren / einfügen kann.

HINWEIS - Für diesen Abschnitt ist die BrightcovePlayer.wsp-Lösungsdatei erforderlich.

  1. Navigieren Sie in der Stammwebsite der Websitesammlung, auf der die Brightcove-App installiert ist, zu Websiteeinstellungen -> Websitesammlungsverwaltungen -> HTML-Feldsicherheit.

  3. Navigieren Sie in der Stammwebsite der Websitesammlung, auf der die Brightcove-App installiert ist, zu Websiteeinstellungen -> Webdesigner-Galerien -> Lösungen.

  7. Stellen Sie sicher, dass die BrightcoveConnectorResources-Dokumentbibliothek während der Installation erstellt wurde.
  8. Wenn nicht, erstellen Sie eine neue Dokumentbibliothek namens BrightcoveConnectorResources
  9. und laden Sie BrightcovePlayer.css und BrightcovePlayer.js in diese Dokumentbibliothek hoch.

Installieren von Proxy Server für die Brightcove-API-Kommunikation

Da der Connector eine clientseitige JavaScript-Anwendung ist, ist ein Proxyserver erforderlich, um die authentifizierte Kommunikation mit der Brightcove-API zu vermitteln. Weitere Informationen finden Sie in der Brightcove-API-Dokument.

Eine .net-Proxy-App ist im Brightcove Video Connect-Paket enthalten. Der Proxyserver sollte an einem über das Internet zugänglichen Speicherort bereitgestellt werden, z. B. in Microsoft Azure. Aus Sicherheitsgründen ist der Proxy so konzipiert, dass nur Verbindungen von Websites zugelassen werden, die in der mit dem Proxy installierten Datei web.config enthalten sind. Der Brightcove: Origins-Schlüssel in der Datei web.config sollte die URL der SharePoint-Website für die Datei enthalten BrightcoveConnector App sowie die Website-URL für die Website, auf der die BrightcovePlayer Komponente wird ausgeführt. Siehe das folgende Beispiel:

<add key="brightcove:origins" value="https://brightcovecms-a3da847a1c9e33.sharepoint.com, https://brightcovecms.sharepoint.com" />

Mehrere Sites können als durch Kommas getrennte Zeichenfolge aufgeführt werden.

Starten von Brightcove Video Connect für SharePoint

Um die Anwendung zu starten, kann der Benutzer auf klicken BrightcoveConnector App in der Liste "Site-Inhalt" oder klicken Sie auf einen Menü-Link, falls dies vom SharePoint-Administrator konfiguriert wurde.

In beiden Fällen wird der Benutzer zur Startseite der Connector-App weitergeleitet. Diese Seite kann vom Client weiter konfiguriert werden, um zusätzliche unternehmensspezifische Nachrichten, zusätzliche Links usw. aufzunehmen. Wie im Installationsabschnitt beschrieben, werden abhängig von der Gruppenmitgliedschaft des Benutzers möglicherweise die Registerkarten "Kontoverwaltung" und "Videos hinzufügen" angezeigt.

Die Hauptnavigation in der linken Spalte des Connectors ist immer vorhanden und ermöglicht es dem Benutzer, durch alle Abschnitte zu navigieren, auf die er Zugriff hat (siehe den obigen Abschnitt zu Berechtigungen). Nachfolgend finden Sie eine kurze Beschreibung des Zwecks jeder Komponente des Connectors:

  • Zuhause - Die Zielseite beim ersten Aufrufen des Connectors.
  • Kontoverwaltung - Bietet die Schnittstelle zum Eingeben der Brightcove Video Cloud-Kontoinformationen in SharePoint.
  • Videos hinzufügen - Laden Sie neue Videos in die Video Cloud hoch. Ein Videoobjekt wird auch in SharePoint erstellt (mit Ausnahme der eigentlichen Videodatei).
  • Videos verwalten - Durchsuchen und bearbeiten Sie vorhandene Videos und importieren Sie Videos aus der Video Cloud in SharePoint.
  • Wiedergabelisten verwalten - Erstellen Sie neue Wiedergabelisten und durchsuchen und verwalten Sie vorhandene Wiedergabelisten.

Es gibt auch Video von Brightcove hinzufügen und Video bearbeiten Komponenten, die in der Hauptnavigation angezeigt werden, wenn diese Vorgänge ausgeführt werden.

Verknüpfen mit Video Cloud-Konten

Im Abschnitt "Kontoverwaltung" von Connector werden die Informationen für jedes der verknüpften Brightcove Video Cloud-Konten eingegeben, auf die auf der Site zugegriffen wird. Brightcove-Konten können nicht aus SharePoint erstellt werden. Sie müssen zuerst in Video Cloud erstellt werden. Die Brightcove-Kontoinformationen müssen in SharePoint eingegeben werden, da sie beim Zugriff auf die Brightcove-API für die Kommunikation mit Video Cloud verwendet werden. Es bietet auch die Möglichkeit, einem Konto spezifische Zugriffsberechtigungen zuzuweisen. Für Unternehmenskunden mit mehreren Konten bietet es die Kontrolle darüber, welche Benutzer auf jedes Konto zugreifen können und ob sie nur Videos durchsuchen oder hochladen können.

Landing Page für die Kontoverwaltung

Auf der Zielseite der Kontoverwaltung wird die aktuelle Liste der konfigurierten Konten zusammen mit den zugewiesenen Gruppen für Autoren und Betrachter angezeigt.

  1. Kontotitel - Derzeit konfigurierte (oder zumindest erstellte) Konten.
  2. Autorengruppe - Die SharePoint-Gruppe hat dem Konto "Autoren" -Rechte zugewiesen.
  3. ViewersGroup - Die SharePoint-Gruppe hat dem Konto "Viewer" -Rechte zugewiesen.
  4. Neues Konto hinzufügen - Öffnen Sie den Bildschirm zur Kontoerstellung, um ein neues Konto einzurichten.

Konto erstellen / bearbeiten

Unabhängig davon, ob Sie ein neues Konto erstellen oder ein vorhandenes Konto bearbeiten, sind der Vorgang und die Optionen identisch. Alle Kontoeigenschaften (mit Ausnahme der Gruppen "Autor" und "Betrachter") müssen zuerst vom Video Cloud-Konto abgerufen werden. Im Folgenden sind die für jedes Konto erforderlichen Eigenschaften aufgeführt.

  1. Kontobezeichnung - Name des Kontos. Dieser Name wird nur in SharePoint verwendet und muss daher nicht mit dem in Video Cloud identisch sein.
  2. DefaultVideoPlayerId - Legt den Standard-Player für Videos fest.
  3. DefaultPlaylistPlayerId - Legt den Standard-Playlist-Player fest.
  4. Konto-ID - Die Brightcove-Konto-ID aus der Video Cloud.
  5. Kunden ID - Die Brightcove API Authentication Client ID.
  6. Kundengeheimnis - Das Geheimnis des Brightcove-API-Authentifizierungsclients.
  7. Temporärer Speicherort - Der temporäre Speicherort, der beim Hochladen von Videos und Textspuren in die Video Cloud verwendet werden soll. Dropdown ermöglicht die Auswahl von Amazon Web Services (AWS) oder Dropbox.
  8. AWS-Zugriffsschlüssel-ID - Bei Verwendung von AWS wird hier die Zugangsschlüssel-ID eingegeben.
  9. AWS Secret Access Key - Bei Verwendung von AWS wird hier der geheime Zugriffsschlüssel eingegeben.
  10. AWS Bucket Name - Bei Verwendung von AWS wird hier der AWS-Bucket-Name eingegeben.
  11. Dropbox-Konto-ID - Bei Verwendung von Dropbox wird hier die Dropbox-Konto-ID eingegeben.
  12. Dropbox-Zugriffstoken - Bei Verwendung von Dropbox wird hier das Dropbox-Zugriffstoken eingegeben.
  13. Benutzerdefinierte Felder - Wird verwendet, um die in Video Cloud für dieses Konto erstellten benutzerdefinierten Felder zuzuordnen. Siehe Sektion Benutzerdefinierte Feldzuordnung für mehr Informationen.
  14. Autorengruppe, Zuschauergruppe - Definitionsgruppen für SharePoint-seitige Berechtigungen für das Konto.
    1. Mitglieder der Autorengruppe (in diesem Fall die SharePoint-Gruppe „Eigentümer“) haben die Möglichkeit, Funktionen im Konto auszuführen, die ein Schreib-Token erfordern, z. B. das Hinzufügen eines Videos, das Bearbeiten und Aktualisieren eines vorhandenen Videos, das Erstellen/Bearbeiten einer Wiedergabeliste usw. Sie können auch alle Lesefunktionen ausführen, z. B. das Durchsuchen und Suchen von Videos und Wiedergabelisten.
    2. Mitglieder der Viewer-Gruppe (in diesem Fall die SharePoint-Besuchergruppe) sind nur für Lesefunktionen wie das Durchsuchen und Suchen von Videos und Wiedergabelisten berechtigt.

Um eine der Eigenschaften in der obigen Kontoliste zu bearbeiten, kann der Benutzer entweder auf den Eigenschaftsnamen oder dessen Wert (falls vorhanden) klicken, um das Popup-Dialogfeld für die Eigenschaft zu öffnen:

Benutzerdefinierte Felder definieren

Beim Einrichten eines Kontos im Brightcove-Connector können benutzerdefinierte Felder, die in Video Cloud erstellt wurden, dem Kontoprofil im Connector hinzugefügt werden, sodass sie in den Bereichen Videos hinzufügen und Videos verwalten angezeigt und ihre Werte gespeichert werden in SharePoint für die Suche. Wenn benutzerdefinierte Felder zugeordnet sind, werden sie im Abschnitt Benutzerdefinierte Felder des Kontoverwaltungsbildschirms zusammen mit der Frage aufgelistet, ob sie Pflichtfelder sind oder nicht:

Um benutzerdefinierte Felder hinzuzufügen, zu löschen oder zu bearbeiten, klicken Sie auf die Schaltfläche "Benutzerdefinierte Felder bearbeiten". Das folgende modale Fenster wird angezeigt, das Listen ALLE der benutzerdefinierten Felder, die für das Konto verfügbar sind. HINWEIS - Diese Liste ist NUR verfügbar, wenn das neue Konto zum ersten Mal gespeichert wird.

  1. Benutzerdefiniertes Feld für die Videowolke - Namen ALLER benutzerdefinierten Felder, die in Video Cloud für das ausgewählte Konto definiert wurden.
  2. Art - Die Art des benutzerdefinierten Feldes (Text oder Liste).

  2. Beschreibung - Eine kurze Beschreibung der Wiedergabeliste.
  3. Art - Die Art der Wiedergabeliste - Manuell oder Smart. Dieses Beispiel zeigt eine manuelle Wiedergabeliste. Das Beispiel im Abschnitt "Bearbeiten einer Wiedergabeliste" zeigt die Optionen für eine intelligente Wiedergabeliste.
  4. Verfügbare Videos - Zeigen Sie bei einer manuellen Wiedergabeliste das Suchfeld und die Liste der im Konto verfügbaren Videos an. Auf diese Weise kann der Benutzer die Videoauswahl eingrenzen, um sie der Wiedergabeliste hinzuzufügen.
  5. Liste der verfügbaren Videos - Liste der verfügbaren Videos im Konto oder Filter basierend auf dem Suchbegriff auf. Ein oder mehrere Videos können durch Klicken auf die Videos ausgewählt werden. Ausgewählte Videos werden grün hervorgehoben. Um die Auswahl eines Videos aufzuheben, klickt der Benutzer erneut darauf. Auf diese Weise können mehrere Videos in einer Aktion in die Wiedergabeliste verschoben werden.
  6. Aktionspfeile - Verschieben Sie ausgewählte Videos in die Wiedergabeliste hinein und aus dieser heraus.
  7. Videos in dieser Wiedergabeliste - Hier werden die aktuellen Videos in der Wiedergabeliste angezeigt.
  8. Speichern - Übernehmen Sie die Änderungen und starten Sie mit dem Erstellen der neuen Wiedergabeliste in Video Cloud. HINWEIS - Wiedergabelisten werden nicht in SharePoint gespeichert.

Bearbeiten einer vorhandenen Wiedergabeliste

Das Bearbeiten einer vorhandenen Wiedergabeliste wird gestartet, indem Sie entweder auf den Namen oder auf den Link "Bearbeiten" in der Wiedergabelistenliste klicken. Dies öffnet die folgende Ansicht, um die Wiedergabelistenoptionen zu bearbeiten. Dieses Beispiel zeigt die Optionen für eine intelligente Wiedergabeliste. Wenn Sie eines der Felder mit Ausnahme der Wiedergabelisten-ID und des Kontos bearbeiten, können Sie Folgendes ändern:

  1. Name - Der Name der Wiedergabeliste.
  2. Beschreibung - Eine kurze Beschreibung der Wiedergabeliste.
  3. Art - Die Art der Wiedergabeliste - Manuell oder Smart. Dieses Beispiel behandelt die Optionen für eine intelligente Wiedergabeliste.
  4. Stichworte - Bei einer intelligenten Wiedergabeliste kann der Benutzer in diesem Feld Kriterien und die Tags angeben, anhand derer die Videos identifiziert werden sollen, die dynamisch in die Wiedergabeliste aufgenommen werden sollen.
    • Das Dropdown-Menü "Kriterien" enthält "Ein oder mehrere enthalten" oder "Alle enthalten".
  5. Auftrag - Auf diese Weise kann der Benutzer festlegen, wie die Reihenfolge der übereinstimmenden Videos festgelegt wird. Die Auswahl umfasst Folgendes:
    • Alphabetisch
    • Aktivierungsdatum (neueste zuerst)
    • Aktivierungsdatum (älteste zuerst)
    • Gesamtspiele
    • Trailing Week-Spiele
    • Startdatum (älteste zuerst)
    • Startdatum (neueste zuerst)
  6. Beschränken auf - Legt die maximale Anzahl von Videos fest, die basierend auf den ausgewählten Kriterien in die Wiedergabeliste aufgenommen werden.
  7. Vorschau - Listet Videos auf, die den Kriterien entsprechen, und zeigen Sie sie im Fenster „Videos in dieser Wiedergabeliste“ an.
  8. Speichern - Übernehmen Sie die Änderungen und starten Sie die Aktualisierung der Wiedergabeliste in Video Cloud. HINWEIS - Wiedergabelisten werden nicht in SharePoint gespeichert.
  9. Wiedergabeliste löschen - Diese Option markiert die Wiedergabeliste als in Video Cloud gelöscht. Die Wiedergabeliste wird nicht mehr in der Connector-Wiedergabelistenliste angezeigt.

Hinzufügen von Videos zu einer SharePoint-Website

Mit dem SharePoint-Connector können Inhaltsautoren Medien auf verschiedene Arten in ihre Websites einfügen. Dieses Kapitel behandelt die verfügbaren Optionen.

Brightcove Player-Webpart für SharePoint On-Premise

Die Brightbox-Player-Webpart-Sandbox-Lösung generiert ein Webpart, das auf einer Site-Seite auf der internen oder externen Website platziert wird, und spielt das Brightcove-Video ab. SharePoint Online schränkt die Möglichkeit für Benutzer ein, benutzerdefinierte Webparts hinzuzufügen, sodass dieser Mechanismus zum Einfügen von Videos nur für lokale SharePoint-Installationen verfügbar ist.

Das Webpart wurde auch um einige der gleichen Funktionen in der Verwaltungskonsole erweitert, einschließlich Kontoauswahl und erweiterte Suche mit Sortieren und Ordnen, um Videos und Wiedergabelisten schneller zu finden.

Um die Player-Webpart-Optionen anzuzeigen, muss sich die SharePoint-Seite im Modus "Bearbeiten" (1) befinden, wodurch die Option zum Bearbeiten des Webparts (2) verfügbar gemacht wird. Die Webpart-spezifischen Optionen finden Sie im Abschnitt "Brightcove-Konfiguration" (3) des Webpart-Editors.

Wenn Sie den Abschnitt "Brightcove-Konfiguration" erweitern, werden die aktualisierten Funktionen der Player-Webpart-Sandbox-Lösung angezeigt.

Auswählen eines Videos, das im Webpart abgespielt werden soll

  1. Brightcove-Konfiguration - Dieser Abschnitt des Webpart-Editors bietet Zugriff auf die Parameter zum Konfigurieren des Webparts und zum Auswählen der abzuspielenden Videos / Wiedergabelisten.
  2. Spielerbreite, Spielerhöhe - Stellen Sie die Abmessungen des Players ein.
  3. Spieler-ID - Geben Sie die ID des zu verwendenden Players an. Es ist wichtig, dass der ausgewählte Player aus demselben Konto stammt wie das ausgewählte Video, wenn es sich um eine Umgebung mit mehreren Konten handelt. Es gibt keine "Auswahl" für den Player, daher muss die ID von Video Cloud manuell eingegeben werden.
  4. Video ID - Das Video-ID-Feld wird mit einer Videoauswahl gefüllt. Wenn der Benutzer auf das Ellipsenfeld rechts neben dem Feld klickt, wird das Auswahlfenster angezeigt.
  5. Auswahlfenster
    • Konto auswählen - Zeigen Sie die Konten an, auf die der aktuelle Benutzer zugreifen darf.
    • Verfügbare Videos - Filtern Sie die Videoergebnisse nach denen, die den Begriff enthalten. Name, Tags und Beschreibungsinformationen sowie SharePoint-Metadatenfelder, sofern definiert, werden durchsucht. Es kann jeweils nur in einem Konto gesucht werden. Cross-Account-Suche wird nicht unterstützt.
    • Sortieren nach und Bestellen nach
    • Video Listing Fenster - Wenn Sie auf ein Video klicken, wird es zum Abspielen ausgewählt. Die Liste ist auf die Anzeige von 30 Videos beschränkt, sodass möglicherweise nicht alle Videos im Konto angezeigt werden, wenn Sie zum ersten Mal ein Konto auswählen. Mithilfe der Suchfunktion können Sie die Liste verfeinern, um das gewünschte Video zu finden.
    • OK-Taste - Übernehmen Sie die Auswahl und füllen Sie das Video-ID-Feld mit dem entsprechenden ID-Wert.

Sobald die Player-ID und die Video-ID festgelegt sind, ist das Webpart konfiguriert und spielbereit!

Auswählen einer Wiedergabeliste zum Abspielen im Webpart

Der Vorgang zum Auswählen einer abzuspielenden Wiedergabeliste ist fast identisch mit dem Auswählen eines Videos.

  1. Brightcove-Konfiguration - Dieser Abschnitt des Webpart-Editors bietet Zugriff auf die Parameter zum Konfigurieren des Webparts und zum Auswählen der abzuspielenden Videos / Wiedergabelisten.
  2. Spielerbreite, Spielerhöhe - Stellen Sie die Abmessungen des Players ein.
  3. Spieler-ID - Geben Sie die ID des zu verwendenden Wiedergabelisten-Players an. Es ist wichtig, dass der ausgewählte Player aus demselben Konto stammt wie das ausgewählte Video, wenn es sich um eine Umgebung mit mehreren Konten handelt, und dass er so konfiguriert ist, dass eine Wiedergabeliste angezeigt wird. Es gibt keine "Auswahl" für den Player, daher muss die ID von Video Cloud manuell eingegeben werden.
  4. Playlist ID - Die Video-ID wird mit einer Videoauswahl gefüllt. Wenn der Benutzer auf das Ellipsenfeld rechts neben dem Feld klickt, wird das Auswahlfenster angezeigt.
  5. Auswahlfenster
    • Konto auswählen - Zeigen Sie die Konten an, auf die der aktuelle Benutzer zugreifen darf.
    • Sortieren nach und Bestellen nach
    • Wiedergabelisten-Auflistungsfenster - Wenn Sie auf eine Wiedergabeliste klicken, wird diese zum Abspielen ausgewählt. Die Liste ist auf die Anzeige von 30 Wiedergabelisten beschränkt, sodass möglicherweise nicht alle Wiedergabelisten im Konto angezeigt werden, wenn Sie zum ersten Mal ein Konto auswählen. Mithilfe der Suchfunktion können Sie die Liste verfeinern, um die gewünschte Wiedergabeliste zu finden.
    • OK-Taste - Übernehmen Sie die Auswahl und füllen Sie das Feld Playlist ID mit dem entsprechenden ID-Wert.

Sobald die Playlist-ID und die Video-ID festgelegt sind, ist das Webpart konfiguriert und kann abgespielt werden!
