    Adobe-AEM-brightcove Anschluss: Konfiguration

    In diesem Thema wird die Konfiguration des Brightcove Connectors für Adobe Experience Manager (AEM) behandelt.

    Einführung

    Der Adobe-AEM-brightcove Connector ermöglicht es Ihnen, Brightcove Video Cloud-Videos und -Playern in AEM zu verwalten und Videos einfach in AEM-Seiten einzubetten.

    Beachten Sie, dass dieser Leitfaden AEM 6.3 verwendet. Der Konnektor funktioniert auch mit AEM 6.2 und AEM 6.4, und die Schritte sind ähnlich.

    Schritte der Konfiguration

    In den folgenden Abschnitten werden Sie durch die Schritte zur Konfiguration des Adobe-AEM-BrightCove Connectors mit der klassischen Benutzeroberfläche geführt. Beachten Sie, dass vor Abschluss dieser Schritte der Connector installiert werden muss - siehe Adobe-AEM-brightcove Connector: Installation

    Deaktivieren Sie DAM-Transco

    Wenn Sie Video-Assets in den AEM-DAM hochladen, wird standardmäßig ffmpeg im Hintergrund ausgeführt, um das Video in eine Vielzahl von Formaten umzukodieren.  Wenn Sie die transcodierten Assets im DAM nicht benötigen, sollten Sie die Transkodierung ausschalten, um Verarbeitungszeit und Speicher zu sparen:

    1. Öffnen Sie die AEM-Tools:
    2. Workflow wählen:
    3. Wählen Sie Modelle:
    4. Wählen Sie die Listenansicht:
    5. Wählen Sie DAM Update Asset:
    6. Wählen Sie Bearbeiten aus:
    7. Doppelklicken Sie im Workflow auf FFmpeg Transcoding, um es zu bearbeiten:
    8. Wählen Sie im Dialogfeld „Schritteigenschaften“ die Registerkarte „Prozess“:
    9. Ändern Sie den Prozess auf „ Keine Operation“:
    10. Ändern Sie Ok um den Dialog zu schließen und dann die Änderungen zu speichern

    Wenn Sie die lokal transcodierten Video-Assets benötigen, müssen Sie sicherstellen, dass ffmpeg auf Ihrem AEM-Server installiert ist:

    1. Stellen Sie sicher, dass Sie FFmpeg installiert haben, indem Sie eine Befehlszeile öffnen und Folgendes eingeben: 
      ffmpeg
    2. Wenn der ffmpeg Befehl nicht gefunden wird, müssen Sie ihn installieren:
      • Auf Mac installieren
      • Unter Windows installieren
      • Unter Linux installieren

    Geben Sie Kontoinformationen

    Jetzt können Sie den AEM-BrigghtCove Connector mit Video Cloud-Kontoinformationen konfigurieren.

    1. Öffnen Sie den AEM Web Console-Konfigurationsmanager (http://localhost:4502/system/console/configMgr)
    2. Suchen Sie im Brightcove Service , um den Service zu finden:
    3. Klicken Sie auf das + Schild um eine neue Konfiguration hinzuzufügen:
    4. Geben Sie im Dialog Werte wie unten gezeigt ein. Anweisungen zum Abrufen einer Client-ID und eines Client-Geheimnisses finden Sie unter Verwalten von API-Authentifizierungsdaten . Die Berechtigungen, die Sie hier benötigen, sind:
      • Im Folgenden finden Sie die empfohlenen Werte für das Konfigurationsformular:
        • Kontoalias: Geben Sie einen Namen für das Konto ein, das im Connector angezeigt werden soll (erforderlich)
        • Konto-ID: Geben Sie Ihre Brightcove-Verlagsnummer ein (erforderlich)
        • Client-ID: Geben Sie Ihre Brightcove-Client-ID auf der Brightcove API-Authentifizierungsseite ein (erforderlich)
        • Client Secret: Geben Sie Ihren Brightcove Client ein Geheime API (erforderlich)
        • Zugelassene Gruppen: Geben Sie die Gruppe an, die auf den Connector zugreift (erforderlich), und stellen Sie sicher, dass es sich um eine Gruppe handelt, in der Sie
        • Spieler-Store-Pfad: Behalten Sie den Standardwert bei oder verwenden Sie den Ordner Ihrer Wahl für Spieler für dieses Brightcove-Konto (erforderlich)
        • Standard-Videoplayer-ID: Behalten Sie den Standardwert bei oder wählen Sie eine andere Spieler-ID (erforderlich)
        • Standard Video Player Key: veraltet, leer lassen oder Wert des vorherigen Konnektors verwenden
        • Standard-Playlist-Spieler-ID: Geben Sie eine für Wiedergabelisten aktivierte Spieler-ID ein, wenn Sie Playlists verwenden möchten (erforderlich)
        • Standard-Playlist-Spielerschlüssel: veraltet, leer lassen oder Wert des vorherigen Konnektors verwenden
        • Proxy-Server: Lassen Sie das Feld leer oder geben Sie einen Proxy ein, wenn Sie zuvor einen Proxy verwendet haben
        • Dam Integration Path: Behalten Sie den Standardwert bei oder verwenden Sie einen Ordner Ihrer Wahl für Video-Assets für dieses Brightcove-Konto (erforderlich)
        • Aufschlussprofil: Geben Sie das Brightcove Ingest-Profil an, das für hochgeladene Videos verwendet wird. Der leere Wert verwendet das Standardprofil. Wenn leer/default nicht funktioniert, bitten Sie den Brightcove-Support, um sicherzustellen, dass ein Standardprofil im Video Cloud-Backend definiert ist
    5. Testen Sie Ihre Konfiguration.  Navigieren Sie zu http://localhost:4502/brightcove/admin und wählen Sie das Brightcove-Konto aus, das gerade im Pulldown in der oberen rechten Ecke hinzugefügt wurde.  Wenn keine Videos angezeigt werden, hat dies wahrscheinlich einen der folgenden Gründe:
      1. Ihre Konto-ID, Ihre API-Anmeldeinformationen, die Client-ID und das Kundengeheimniswurden möglicherweise falsch in die Konnektorkonfiguration eingegeben
      2. Ihre API-Anmeldeinformationen wurden in Brightcove nicht ordnungsgemäß konfiguriert
      3. Der aktuell angemeldete Benutzer ist kein Mitglied einer der zulässigen Gruppen in der Konnektorkonfiguration
      4. Ihre Netzwerkkonfiguration blockiert den API-Zugriff auf die Cloud-Server von Brightcove. Weitere Informationen finden Sie unter „Debuggen von Netzwerk- und API-Verbindungsproblemen“ .
    6. Wenn Sie mehrere Video Cloud-Konten verwenden, wiederholen Sie die vorherigen Schritte für jedes Konto.

    Richten Sie Brightcove Spieler ein

    Die letzte Konfigurationsaufgabe besteht darin, Brightcove Players einzurichten.

    1. Navigieren Sie zu den Site-Admin-Tools (http://localhost:4502/siteadmin#/content in der klassischen Benutzeroberfläche oder http://localhost:4502/sites.html/content in der Touch-Benutzeroberfläche)
    2. Erstellen Sie im Ordner Websites der obersten Ebene einen neuen Ordner, der dem Players Store Path in der Konfiguration entspricht.  Die Standardeinstellung ist ein Ordnername in der Konfiguration heißt brightcovetools mit einem Ordner namens Player.  Wenn Sie eine Verbindung zu mehreren Brightcove-Konten herstellen, möchten Sie möglicherweise für jedes Konto einen eindeutigen Spielerordner erstellen.
    3. Erstellen Sie im Spieler-Ordner eine neue Brightcove Player Page:
    4. Nachdem Sie die Seite erstellt haben, klicken Sie mit der rechten Maustaste darauf und wählen Sie im Menü der klassischen Benutzeroberfläche Eigenschaften aus. Wählen Sie in der Touch-Benutzeroberfläche die Seite aus und klicken Sie dann auf Eigenschaften.
    5. Gehen Sie auf die Registerkarte Brightcove Basic des Dialogs und das Brightcove-Konto und wählen Sie dann einen Spieler aus der Liste aus:
    6. Wenn du auch mehrere Spieler für die Veröffentlichung zur Verfügung stellen möchtest, wiederhole die obigen Schritte, um eine Seite für jeden Spieler zu erstellen.
    7. Wenn du auch Wiedergabelisten verwenden möchtest, wiederhole die obigen Schritte, um eine Seite für einen Playlist-Player zu erstellen.
    8. Wenn Sie mehrere Brightcove-Konten verwenden, müssen Sie für jedes einzelne Spielerseiten erstellen.

    Metadatenschema für den Standort von Brightcove Asset aktivieren

    Wenn Sie Touch UI mit dem v5.4-Connector oder höher verwenden, müssen Sie das Metadatenschema für den Brightcove-Asset-Standort aktivieren.   Dies ist nicht notwendig, wenn Sie nur die klassische Benutzeroberfläche verwenden.  Aktivieren Sie das Metadatenschema mit den folgenden Schritten:

    1. Öffnen Sie die AEM-Tools:
    2. Wählen Sie Assets:
    3. Wählen Sie Metadatenschemata:
    4. Wählen Sie Brightcove aus und klicken Sie auf Auf Ordner anwenden:
    5. Wenn der brightcove_assets -Ordner nicht existiert, erstellen Sie einen Ordner mit dem Namen brightcove_assets:
    6. Geben Sie den Namen „brightcove_assets“ ein:
    7. Wählen Sie brightcove_assets aus und klicken Sie auf Übernehmen:

    Sie haben jetzt die Konfiguration des AEM-BrigghtCove Connectors abgeschlossen. Fahren Sie mit Verwenden des Connectorsfort.
