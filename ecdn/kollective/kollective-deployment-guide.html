    Kollective eCDN Bereitstellungsleitfaden

    In diesem Thema erfahren Sie, wie Sie den Kollective eCDN in den Brightcove Player integrieren.

    Übersicht

    Brightcove hat eine Integration ins Leben gerufen, um Brightcove Players mithilfe des Kollective eCDNVideo On Demand (VOD) und Brightcove Live-Events zur Verfügung zu stellen.

    Eine eCDN-Lösung hilft bei der Bereitstellung von Inhalten an Unternehmensnetzwerke. Sie lindern Netzwerkengpässe, wenn eine große Anzahl gleichzeitiger Zuschauer dieselben Inhalte beobachtet, z. B. in Unternehmensbüros. Mit eCDN laden Sie Videoinhalte für jede eCDN-Server-Instance einmal vom Original-Server herunter. Zuschauer im Unternehmensnetzwerk erhalten die Inhalte vom eCDN-Server über das LAN.

    Einige der Vorteile der Verwendung eines eCDN umfassen Folgendes:

    • Reduzierter WAN-Verkehr für Unternehmensstandorte
    • Verbessertes Fernseherlebnis im großen
    • Reduzierte Netzwerkkosten
    • Erhöhte Sicherheit, da der eCDN hinter der Unternehmensfirewall steht
    • Performance-Überwachung

    Anforderungen

    Die folgenden Anforderungen gelten für das eCDN-Plugin:

    • Sie müssen einen Brightcove Player Version 6.42.2+ verwenden
    • Videos müssen für Dynamic Delivery aufgenommen werden

    Erste Schritte

    Um das kollective-Plugin zu verwenden, benötigen Sie eine Ersteinrichtung:

    1. Bereitstellen von serverseitigen Anwendungen
    2. Installieren des Player-Plugins
    3. Whitelisting von Domainnamen

    4. Nachdem Sie die Bereitstellungsschritte abgeschlossen haben, können Sie Ihre Inhalte synchronisieren und den Player-Code einbetten. Einzelheiten finden Sie unter den folgenden Themen:

      • Benutzerhandbuch für kollective eCDN

    Bereitstellen von serverseitigen Anwendungen

    Die Backend-eCDN-Anwendung muss auf Ihrem Server bereitgestellt werden.

    Anforderungen an Server

    Die folgenden Anforderungen gelten für den Hosting-Server für die Backend-Anwendung.

    • NodeJS Version 13+
    • Hosting-Server: Linux- oder Windows-Server (Die Backend-Anwendung kann auch auf Heroku bereitgestellt werden)
    • MongoDB Datenbank

    Die MongoDB kann sich auf demselben Server wie der Hosting-Server, auf einem anderen Server oder mit einer Cloud-Datenbank des MongoDB Atlas befinden.

    Einstellen von Konfigur

    Sie nehmen ein paar Konfigurationsänderungen für die Backend-Anwendung vor. Befolgen Sie diese Schritte:

    1. Öffnen Sie im Stammverzeichnis für das Backend-Anwendungsbereitstellungspaket die Datei config.env .

    2. Produktionsmodus - Legen Sie den Modus für die Ausführung der Anwendung in der Produktion oder Entwicklung fest.

      Entweder 

      NODE_ENV=production

      oder 

      NODE_ENV=development

    3. Application Port - Legen Sie den Port fest, auf dem die Anwendung ausgeführt werden soll.

      Beispiel: 

      PORT=3000

    4. Datenbank - Die Backend-Anwendung benötigt eine MongoDB-Datenbank.

      Sie können die Datenbankverbindung im folgenden Format angeben: 

      DATABASE=mongodb://{host}:{port}/{databaseName}

      Beispiel: 

      DATABASE=mongodb://localhost:27017/Brightcove

      Wenn Sie eine Cloud-Datenbank von MongoDB Atlas verwenden möchten, können Sie sie wie folgt einstellen: 

      DATABASE=mongodb+srv://{user}:{password}@{cluster}/{databaseName}?retryWrites=true&w=majority

      Beispiel: 

      DATABASE=mongodb+srv://myuser:mypassword@cluster0-xafsz.mongodb.net/MyDatabase?retryWrites=true&w=majority

    5. E-Mail-Service - Die Backend-Anwendung erfordert eine E-Mail-Service-Einstellung zum Senden von Benachrichtigungs-E-Mails.

      SendGrid

      Um E-Mails mit dem SendGrid-E-Mail-Dienst zu senden, verwenden Sie Folgendes mit Ihrem SendGrid-Benutzernamen und Passwort: 

      # Send email using Sendgrid email service.
EMAIL_SERVICE=SendGrid

# Send email using Sendgrid email service.
SENDGRID_USERNAME=your SendGrid username
SENDGRID_PASSWORD=your SendGrid password
      Sonstiges

      Um E-Mails mit anderen E-Mail-Diensten zu senden, verwenden Sie Folgendes mit Ihrem E-Mail-Serverhost, Port und Benutzernamen, Passwort: 

      # Set EMAIL_SERVICE=Other to send email using normal email service using host and port
EMAIL_SERVICE=Other

# Send email using host and port
EMAIL_HOST=your email host
EMAIL_PORT=your email port
EMAIL_USERNAME=your email username
EMAIL_PASSWORD=your email password
EMAIL_FROM=your email from address

    6. Kollective Publish API Basis-URL - Die serverseitige Anwendung ruft die Kollective veröffentlichen API auf, um ein Video von Brightcove mit Videoquellen an Kollective zu veröffentlichen und ein Content-Token zurückzuholen. Hier geben Sie die Basis-URL der Kollective Publish API an: 

      # Kollective publish API base url
KOLLECTIVE_PUBLISH_API_BASE_URL=https://content.stage.kollective.app/api

    7. Brightcove API Base-URLs - Die serverseitige Anwendung benötigt die folgenden Brightcove API-Basis-URLs: 

      # Brightcove API base URLs
BRIGHTCOVE_ACCESS_TOKEN_BASE_URL=https://oauth.brightcove.com/v4/access_token
BRIGHTCOVE_CMS_API_BASE_URL=https://cms.api.brightcove.com/v1
BRIGHTCOVE_LIVE_API_BASE_URL=https://api.bcovlive.io/v1

    8. Benutzerdefinierte Feldnamen — Die folgenden Einstellungen für die Namen benutzerdefinierter Felder können nicht geändert werden: 

      # Required custom field names for each Brightcove Account ID
CUSTOM_FIELD_KOLLECTIVE_CONTENT_TOKEN=kollective_content_token
CUSTOM_FIELD_KOLLECTIVE_IS_SYNCHRONIZED=kollective_is_synchronized
CUSTOM_FIELD_KOLLECTIVE_SYNCHRONIZED_TIME=kollective_synchronized_time
CUSTOM_FIELD_KOLLECTIVE_LAST_SYNC_STATUS=kollective_last_sync_status

    Hinzufügen von benutzerdefinierten Feldern

    Als Nächstes können Sie die erforderlichen benutzerdefinierten Felder in Video Cloud Studio definieren.

    1. Erweitern Sie in Video Cloud Studio das Dropdown-Menü Admin und wählen Sie Videofelderaus.

    2. Wählen Sie auf der Seite „Videofelder“ die Schaltfläche „ Benutzerdefiniertes Feld hinzufügen “.

    3. Hier fügen Sie benutzerdefinierte Felder hinzu, die für Kollective erforderlich sind.

      Jede Brightcove-Konto-ID, die für ein Syndikatorkonto verwendet wird, benötigt die folgenden benutzerdefinierten Feldnamen, die für die Synchronisierung definiert sind:

      Zusammenfassung der Brightcove Beacon-Funktion
      Name des benutzerdefinierten Feldes Art
      kollective_content_token Text
      kollective_is_synchronisiert Text
      kollective_synchronized_time Text
      kollective_last_sync_status Text

      Geben Sie auf der Detailseite des Videofelds Folgendes ein:

      • Anzeigename - Geben Sie die Namen der benutzerdefinierten Felder aus der obigen Tabelle ein. Speichere jedes Feld separat.
      • Interner Name - Dieser wird aus dem Anzeigenamen generiert.
      • Belassen Sie den Typ als Text
      • Geben Sie eine Beschreibung
      • Lassen Sie die Option Dieses Feld erforderlich machen NICHT ausgewählt

    Bereitstellen der Anwendung

    Nachdem Sie alle erforderlichen Konfigurationsänderungen vorgenommen und die erforderlichen benutzerdefinierten Felder hinzugefügt haben, können Sie die Backend-Anwendung auf Ihrem Server bereitstellen.

    1. Kopieren Sie das serverseitige Anwendungspaket in den Ziel-Bereitstellungsordner auf Ihrem Hosting-Server.

    2. Öffnen Sie eine Konsolenanwendung und wechseln Sie im vorherigen Schritt zum Stammordner des Anwendungspakets. Führen Sie den folgenden Befehl aus, um alle erforderlichen Node-Pakete zu installieren: 

      # npm install --production

    3. Führen Sie den folgenden Befehl aus, um den Standard-Superadmin-Benutzer zu importieren: 

      # npm run import-users

    4. Um die Backend-Anwendung zu starten, führen Sie den folgenden Befehl aus: 

      # npm run start

    5. Wenn alles korrekt eingerichtet ist, können Sie die Anwendung in einem Browser mit der konfigurierten Adresse öffnen.

      Beispiel: 

      http://localhost:3000/login

    6. Melden Sie sich bei der Anwendung an, wenn der oben importierte Standard-Super-Admin-Benutzer verwendet

      • Benutzername: super-admin@brightcove.com
      • Kennwort: br1GHTC0VE

    Installieren des Player-Plugins

    Um Kollective in den Brightcove Player zu integrieren, füge das Kollective-Plugin deinem Player hinzu.

    Das Kollective-Plugin wird von der Brightcove Player-Version v6.45+ unterstützt.

    In den folgenden Schritten wird beschrieben, wie Sie das Plugin mit Video Cloud Studio laden.

    1. Öffnen Sie in Video Cloud Studio das PLAYSER-Modul und erstellen Sie entweder einen neuen Player oder suchen Sie den Player, dem Sie das Plugin hinzufügen möchten.
    2. Wählen Sie den Link für den Spieler aus, um die Eigenschaften des Spielers zu öffnen.
    3. Wähle im linken Navigationsmenü Plugins aus.

    4. Wählen Sie als Nächstes die Schaltfläche Plugin hinzufügen und dann Brightcove Pluginaus.

    5. Erweitern Sie das Dropdown-Menü Brightcove Plugin und wählen Sie kollectiveaus.

    6. Für die Optionen (JSON): Geben Sie Folgendes ein:

      • ksdkUrl - Verweist auf die ksdk Bibliothek von Kollective — verwenden Sie die Version Ihrer Wahl 
      {
    "ksdkUrl": "ksdk URL”
}
    7. Wählen Sie die Schaltfläche „ Speichern “.
    8. Veröffentliche deinen Spieler.
    9. Aktualisieren/ändern Sie den Einbettungscode von data-video-id auf data-kollective-video-id.
      10. Wenn Sie einen Player zur Verwendung mit der Kollective-Integration einbetten, müssen Sie den Einbettungscode von aktualisieren / ändern Daten-Video-ID zu Daten-Kollektiv-Video-ID.

    Whitelisting von Domainnamen

    Kollective muss Ihre Domainnamen zu ihrer Whitelist hinzufügen, damit kein CORS-Fehler auftritt. Geben Sie Kollective die folgenden Domainnamen an:

    • Der Domänenname, in dem Sie Ihre serverseitige Anwendung bereitgestellt haben

    • Die Domain, in der Sie Ihre Webseiten bereitgestellt haben

    Wenn Sie beispielsweise eine Seite unter https://my-server.com/my-video-page haben, die Brightcove Player mit dem Kollective-Plugin verwendet, muss die Domäne my-server.com zur Kollective-Whitelist hinzugefügt werden. Dies ermöglicht die Wiedergabe auf der Seite ohne Rückgabe eines CORS-Fehlers.
