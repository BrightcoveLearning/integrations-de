    Erste Schritte mit Ramp Multicast+ und Video Cloud

    Dieses Thema bietet einen Überblick über die Verwendung der Ramp AltitudeCN™ Multicast+ eCDN Lösung mit Brightcove Video Cloud.

    Einführung

    Rampe AltitudeCDN ™ Multicast + Das Produkt ist ein Live-Videoverteilungssystem für Unternehmen das nutzt Multicast-fähige Netzwerke zu Reduzieren Sie den Videoverkehr über das lokale Netzwerk, das Internet-Gateway und Verbindungen mit begrenzter Bandbreite innerhalb des Unternehmensnetzwerks. Mit Multicast+ ziehen Client-Desktop-Computer, die den Brightcove Player verwenden, den Stream von einem Multicast+-Empfänger-Client auf dem lokalen Computer, der den vom Multicast+-Server im Netzwerk bereitgestellten Live-Stream „abstimmt“.  Der Multicast+-Server zieht den Livestream vom Brightcove CDN oder einem lokalen Streaming-Server. Multicast+ ist eine vollständig lokale Lösung, die als Software-Appliance bereitgestellt wird und von der Managementkonsole von AltitudeCDN™ Altimeter verwaltet wird.  Multicast+ kann eigenständig oder zusammen mit altitudeCN™ Omnicache für gemischte Unicast/Multicast-Netzwerke verwendet werden.

    Die Integration mit dem Brightcove verwendet das Multicast+ Video.js Plug-In , um die Stream-URL auf den lokalen Multicast+-Empfänger-Clientserver umzuschreiben.  Vollständige Details zum Einrichten des Multicast+-Servers für die Verwendung mit Video Cloud finden Sie in dieser TechNote von Ramp.

    Bevor Sie mit den folgenden Anweisungen fortfahren, schließen Sie bitte den Multicast+-Quickstart-Prozess mit dem Ramp-Team ab, um sicherzustellen, dass der Multicast+-Server, die lokalen Multicast+-Empfänger und das lokale Netzwerk ordnungsgemäß konfiguriert sind.

    Spieler-Konfiguration

    Beginnen Sie mit der Konfiguration eines Brightcove Players für die Verwendung des Multicast+ Plug-Ins. Erstelle einen neuen Spieler mit dem gewünschten Skin und den gewünschten Optionen.

    Fügen Sie dann das Multicast+ Video.js Plug-In hinzu.

    Kopieren und einfügen von hier:

          //livetools.ramp.com/omnicache/plugin/multicastplugin.allinone.min.js

    Legen Sie schließlich die Multicast+-Plug-in-Optionen fest.  Zunächst muss der Spieler die Multicast-IP-Adresse und den Port für die Live-Multicast-Sendung kennen.  Der Player kann für eine bestimmte Multicast-Adresse gesperrt werden, indem ein „maddr“ -Parameter in den Optionen wie

          "maddr": "239.0.0.1",

    Im Idealfall erhält der Player die Multicast-Adresse jedoch von dem Brightcove-Medienobjekt, das er gerade abspielt.  Dies ist besonders nützlich, wenn der Spieler vom CDN auf einen Omnicache-Server oder den Stream zurückgreifen muss, da er bereits die ursprüngliche Stream-URL hat.  Wenn in den Plug-in-Optionen nicht „maddr“ angegeben ist, sucht das Plug-In in einem benutzerdefinierten Feld mit dem Namen „multicastaddress“ nach der Multicast-IP-Adresse. Um den Wert Custom Field für ein Medienobjekt festzulegen, öffnen Sie das Medienobjekt in Brightcove Studio und klicken Sie im Abschnitt Benutzerdefinierte Felder auf Bearbeiten:

    Verwenden Sie für eine gemischte Unicast/Multicast-Umgebung den Parameter „omnicache“ in den Optionen, um einen Omnicache-Server anzugeben, auf den der Player zurückgreifen kann, wenn der Multicast+-Empfänger-Client auf dem Wiedergabegerät nicht verfügbar ist oder wenn der Multicast-Stream in diesem Segment des netzwerk.

    Kopieren und einfügen von hier:

    { 
      "params": {
        "minTime1": 15,
        "minTime0": 5,
        "keyStoreUri": "default",
        "receiverType": "native",
        "receiverHost": "hostname",
        "allowHttpFallback": false,
        "omnicache": "https://omnicache_backup_server"
      }, 
      "verbose": true
    }

     

    Parameter-Definitionen:

    • maddr፦ Die Multicast-Stream-Adresse und der Port.  Dieses Plug-In kennt auch das benutzerdefinierte Feld „Multicastaddress“, das verwendet werden kann, anstatt das Maddr im Player-Plug-In fest zu kodieren.
    • AllowHttPfAllback: (Standard false) - verhindert, dass der Code versucht, eine Verbindung zu einem unsicheren Empfänger herzustellen, wenn er sich auf einer sicheren Seite befindet (in einigen Situationen möglich)
    • keystoreURI: (Standard „Standard“) - erlaubt dem Code, einen eingebetteten Schlüsselspeicher zu verwenden, anstatt nach einem zu ladenden zu suchen
    • ReceiverHost: „brightcove.multicast-receiver.altitude.net“ - liefert den zertifizierten Namen für localhost
    • ReceiverType: „native“ - ermöglicht es dem Code, Standardeinstellungen zu verwenden, anstatt zu versuchen, eine externe Datei zu finden, die für das Betriebssystem spezifisch ist
    • minTime0: begrenzt die Abfrage des Empfängers auf die angegebene Anzahl von Sekunden empfangener 0-Bytes
    • minTime1: begrenzt die Abfrage des Empfängers auf die angegebene Anzahl von Sekunden, in denen das erste Segment von 3 nicht empfangen wurde
    • Omnicache: URL für Omnicache-Backup-Server, wenn der Multicast-Stream nicht gefunden werden kann

    Brightcove Player einbetten

    Schließlich betten Sie den Brightcove Player mithilfe des Standard-Player-Veröffentlichungsprozesses auf Ihrer Webseite ein.


    Bereitstellen von Live Stream-Manifest für Multicast+

    Um einen Live-Stream im Unternehmensnetzwerk zu übertragen, muss Multicast+ die vom Brightcove CDN oder einem internen Streaming-Server bereitgestellte Stream-Manifest-URL kennen. Wenn Sie den Live-Workflow in Brightcove Video Cloud verwenden, ist die URL des Streammanifests erst bekannt, wenn das Live-Ereignis mit dem Streaming beginnt. Um die Manifest-URL für Multicast+ bereitzustellen, starten Sie den Live-Stream, kopieren das Streammanifest und speichern dann das Stream-Manifest über die Altimeter-Konsole auf dem Multicast+-Server.

    Starten Sie den Livestream und bestimmen Sie das Stream-

    Um das Stream-Manifest zu bestimmen, erstellen Sie zuerst das Live-Ereignis. Beachten Sie, dass Sie ein Live-Event Tage oder Wochen vor dem Start des Live-Streams erstellen können. In diesem Fall werden Sie zuerst einen Teil der folgenden Anweisungen ausfüllen und später den Rest der Anweisungen abschließen.

    Wählen Sie den Standardplayer, nicht den Multicast+-fähigen Player aus.

    Wähle dein Livestream-Profil.

    Stellen Sie sicher, dass Ihre Ausgabeoptionen keine RTMP-formatierten Formatvarianten haben, da es einfacher ist, die URL des HLS-Streammanifests zu finden.

    Starten Sie den Encoder für den Livestream und kopieren Sie die Manifest-URL. Wenn Sie anstelle des QR-Codes wie unten gezeigt einen Live-Event-Vorschau-Player sehen, haben Sie immer noch eine RTMP-formatierte Wiedergabe.

    Wenn Sie RTMP-Wiedergaben in Ihrem Live-Stream behalten müssen, können Sie die Stream-Wiedergabe jederzeit starten und die Manifest-URL vom Browser-Debugger abrufen. Kopieren Sie dazu die Player-URL.

    Fügen Sie die Player-URL in eine neue Registerkarte im Browser ein, öffnen Sie den Browser-Debugger (in diesem Beispiel gezeigt Chrome) und kopieren Sie die URL des HLS-Stream-Manifests master.m3u8.

    Befolgen Sie schließlich die Anweisungen in dieser Multicast+TechNote oder einer anderen Multicast+- und Altimeter-Dokumentation, um einen neuen Broadcast mit der Stream-URL zu konfigurieren.
