WordPress-Brightcove-Connector: Häufig gestellte Fragen

Dieses Thema enthält häufig gestellte Fragen zum Wordpress-Brightcove-Connector.

Kann ich das Plugin auf einer WordPress-Installation ausführen, die nicht öffentlich zugänglich ist?

Ja, es funktioniert unabhängig davon, ob es öffentlich ist oder nicht. Funktionen, die einen Rückruf auf Ihre Website erfordern, wie z. B. Statusaktualisierungen, das Hochladen von Videos usw., sind jedoch nicht verfügbar.

 Gibt es Filter für Plugin- / Theme-Entwickler? 

brightcove_account_actions = [bearbeiten, löschen]; // Welche Aktionen stehen beim Bearbeiten einer Brightcove-Quelle zur Verfügung?

Funktioniert das auf Multisite?

Ja, es wird.

Kann ich mehr als ein Brightcove-Konto verwenden?

Ja, Sie können Quellen aus vielen Brightcove-Konten hinzufügen, wenn Sie möchten.

Kann ich verschiedene Berechtigungen für Benutzer festlegen?

Ja. Eine Matrix mit Benutzerberechtigungen finden Sie in dieses Dokument. Um die Benutzerrollen zu ändern, empfehlen wir die Verwendung der WordPress-Funktion add_cap: https://codex.wordpress.org/add_cap

Wie funktioniert die Synchronisierung?

Das Plugin ruft einfach Informationen direkt von der Brightcove-API zur Anzeige ab und synchronisiert keine Videos lokal.

Wie kann ich die maximale Größe der Upload-Datei erhöhen?

Die maximale Dateigröße wird von Ihrem Webserver und Ihrer PHP-Konfiguration bestimmt. Sie müssen den Wert von upload_max_filesize und post_max_size in Ihrer php.ini festlegen. Mit php_ini_loaded_file () können Sie herausfinden, wo sich Ihre PHP.ini befindet.

Wie aktiviere ich die Änderungsverfolgung, damit ich sehen kann, wer ein Video aktualisiert hat und wann sie es erstellt haben?

Erstellen Sie ein benutzerdefiniertes Feld vom Typ „Text' mit dem internen Namen „_change_history“. Bei jeder Aktualisierung eines Videos werden der Benutzername und die aktuelle Uhrzeit zu einer Liste der in diesem Feld aufgezeichneten Änderungen hinzugefügt.
